Introduccion

En nuestro proyecto formativo, hemos estado desarrollando un software de control de inventario de ambienes TIC. Este software esta diseñado para ser utilizado por los celadores encargados de los ambientes y encargados de gestionar y supervisar el estado de las salas de informatica dentro del complejo del SENA Yamboró.

El objetivo de este software es brindar una herramienta facil de usar y eficiente para mantener un seguimiento preciso del inventario de los equipos y suministros de los ambientes.

Tambien se ha trabajado en la definicion de las especificaciones del software, incluyendo su arquitectura, caracteristicas, funcionalidades, etc. Tambien he realizado un analisis de precios para asegurar que el software sea accesible y rentable para el complejo del SENA.

Software

A continuacion describo algunas especificaciones tecnicas del software del subproyecto formativo:

Gestión de inventario: El software debe permitir la gestión completa del inventario de las salas de informática, incluyendo equipos como computadoras,monitores, teclados, ratones, entre otros, así como suministros como papel, cartuchos de tinta, cables, entre otros.

Registro de entradas y salidas: El software debe permitir el registro de las entradas y salidas de equipos y suministros de las salas de informática, parallevar un seguimiento detallado de su estado, ubicación y disponibilidad.

Alertas y notificaciones: El software debe ser capaz de enviar alertas y notificaciones al personal encargado del mantenimiento y la gestión de las salas deinformática, cuando un equipo o suministro necesita ser reemplazado, reparado o renovado.

Informes y estadísticas: El software debe permitir la generación de informes y estadísticas sobre el estado del inventario de las salas de informática, paraayudar en la toma de decisiones y en la planificación de recursos.

Accesibilidad y seguridad: El software debe ser accesible para los celadores encargados de su uso, a través de una interfaz de usuario intuitiva y fácil deusar. Además, debe contar con medidas de seguridad para proteger la información y los datos almacenados en el software.

Integración con otras herramientas: El software debe ser capaz de integrarse con otras herramientas de software y hardware, como sistemas de gestión de ticketso sistemas de monitoreo de red, para mejorar su funcionalidad y eficiencia.

Hardware

A continuacion describo algunas especificaciones tecnicas del hardware del subproyecto formativo:

Computadoras: Las computadoras deben tener un procesador de al menos 2 GHz, memoria RAM de al menos 4 GB y disco duro de al menos 500 GB. Además, deben tener una tarjeta de red para conexión a la red local y una pantalla de al menos 15 pulgadas.

Servidor: Se debe contar con un servidor para alojar la base de datos del software. El servidor debe tener un procesador de al menos 2 GHz, memoria RAM de al menos 8 GB y disco duro de al menos 1 TB.

Red: Se necesita una red local (LAN) que permita la comunicación entre las computadoras y el servidor. La red debe ser confiable y tener una velocidad de transferencia de al menos 100 Mbps.

Impresoras: Es posible que se requiera el uso de impresoras para la impresión de etiquetas de identificación para los equipos y suministros, por lo que se necesitan impresoras con una resolución adecuada para la impresión de etiquetas.

Escáneres de códigos de barras: Para agilizar el proceso de registro de los equipos y suministros, es posible que se necesiten escáneres de códigos de barras que permitan la identificación rápida y precisa de los equipos y suministros.

Sistemas de alimentación ininterrumpida (UPS): Se recomienda el uso de UPS para proteger los equipos y asegurar que el software esté siempre disponible, incluso en caso de cortes de energía.

Licencias

Por lo general, para desarrollar un software es importante tener en cuenta las siguientes licencias:

Licencia de desarrollo: Esta licencia es necesaria para utilizar las herramientas y programas de desarrollo de software, como editores de código, compiladores, librerías, entre otros.

Licencia de código abierto: Si se utiliza código abierto en el software, es necesario tener en cuenta las licencias de los proyectos de código abierto que se estén utilizando. Por ejemplo, si se utiliza la biblioteca de código abierto de jQuery, es necesario tener en cuenta la licencia de jQuery.

Licencia de software propietario: Si se utiliza software propietario en el desarrollo del software, es necesario obtener las licencias correspondientes. Por ejemplo, si se utiliza una base de datos comercial, es necesario tener la licencia correspondiente.

Licencia de distribución: Si se planea distribuir el software, es necesario tener una licencia de distribución que permita la distribución del software y defina los términos y condiciones para su uso.

Capacitaciones

Es importante que los celadores encargados de utilizar el software de control de inventario de ambientes de salas de informática reciban una capacitación adecuada para que puedan administrar el servicio de manera efectiva. Algunos temas que podrían abordarse en la capacitación incluyen:

Introducción al software: Los celadores deben estar familiarizados con el software y comprender su propósito y funcionalidad.

Procedimientos operativos: Los celadores deben aprender los procedimientos operativos que se deben seguir para utilizar el software de manera efectiva. Esto puede incluir cómo registrar equipos y suministros en el sistema, cómo realizar búsquedas de inventario y cómo generar informes.

Seguridad: Es importante que los celadores estén informados sobre la seguridad de los datos y las medidas de seguridad que se han implementado en el software para proteger la información confidencial.

Mantenimiento del software: Los celadores deben aprender cómo realizar el mantenimiento básico del software, como la actualización de versiones y la solución de problemas básicos.

Soporte técnico: Es importante que los celadores sepan cómo solicitar soporte técnico en caso de problemas más complejos o si necesitan asistencia adicional.

La capacitación puede llevarse a cabo mediante sesiones de entrenamiento en persona, manuales de usuario detallados y videos explicativos. También es recomendable establecer un proceso de retroalimentación para que los celadores puedan brindar comentarios sobre el software y la capacitación y así mejorar continuamente el proceso.

Metodologias

Existen diferentes metodologías que se pueden utilizar para desarrollar un software de control de inventario de ambientes de salas de informática, aquí te presento algunas posibles opciones:

Metodología cascada: Esta metodología implica una secuencia de fases lineales, desde la planificación hasta la implementación y el mantenimiento del software. Se requiere que cada fase se complete antes de avanzar a la siguiente, lo que puede garantizar una buena calidad, pero puede ser difícil volver atrás en caso de errores o cambios.

Metodología ágil: La metodología ágil es un enfoque iterativo e incremental que se enfoca en la colaboración y la retroalimentación constante del cliente. Los desarrolladores trabajan en ciclos cortos de entrega de software, conocidos como sprints, y cada sprint incluye una planificación, diseño, desarrollo, pruebas y entrega de un producto parcial.

Metodología DevOps: DevOps combina las metodologías de desarrollo y operaciones para mejorar la colaboración, la entrega continua y la automatización de procesos. Los desarrolladores trabajan en estrecha colaboración con los equipos de operaciones para automatizar los procesos de entrega de software, lo que puede reducir los tiempos de entrega y mejorar la calidad del software.

Metodología en espiral: Esta metodología implica la repetición de un proceso de desarrollo en un ciclo continuo y evolutivo. Cada ciclo incluye una evaluación de riesgos y un análisis de requisitos, diseño, implementación y pruebas, lo que permite un enfoque más flexible y adaptable al cambio.
